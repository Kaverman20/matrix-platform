# Журнал решений (Decision log)

Здесь фиксируем нетривиальные проблемы и принятые решения: **с чем столкнулись →
почему так произошло → как починили**. Цель — чтобы команда понимала контекст,
а не разбиралась заново. Новые записи добавляем сверху.

---

## 2026-06-15 — Кэш Caddy: деплой не подхватывался браузером

**Проблема.** После деплоя фронта браузер продолжал грузить старый JS-бандл —
проверки шли на устаревшем коде, фиксы «не работали».

**Причина.** Caddy отдавал `index.html` без заголовка `Cache-Control`, браузер
кэшировал HTML по эвристике и ссылался на прежний хэшированный бандл.

**Решение.** В `Caddyfile` (на сервере `/opt/synapse-test/`) для сайта
`chat.foxhound.run`:
- `@assets path /assets/*` → `Cache-Control: public, max-age=31536000, immutable`
  (имена файлов содержат хэш — можно кэшировать вечно);
- `@notassets not path /assets/*` → `Cache-Control: no-cache`
  (`index.html` всегда ревалидируется → новый деплой подхватывается сразу).

Деплой Caddy: правка → `caddy validate` → `caddy reload` (graceful, прокси Synapse
не затрагивается). Caddyfile пока живёт только на сервере, в репозитории не отражён.

## 2026-06-15 — Мигание интро в пустом личном чате + бесконечная пагинация

**Проблема.** При входе в пустой DM блок «Имя / Начало личного чата» постоянно
мигал; в фоне шёл шквал запросов пагинации. Замер: ~700k появлений/исчезновений
интро за 5 секунд.

**Причина.** Synapse для DM отдаёт backwards-токен пагинации даже в начале
комнаты. Эффект «заполнить экран историей» считал, что история есть, грузил
страницу без новых сообщений (`paginateBackwards` → `false`) и ставил
`atStart = true`; другой эффект видел, что токен ещё есть (`hasOlder = true`), и
откатывал `atStart = false`. Бесконечный пинг-понг → `RoomIntro` то монтируется,
то нет.

**Решение.** Ввели ref `reachedStart`: как только пагинация вернула «нет новых
сообщений», фиксируем достижение начала, и sync-эффект больше не сбрасывает
`atStart`. Дополнительно `loadOlder` обёрнут в `useCallback` (меньше лишних
перерендеров). Файлы: `apps/surf-chat-web/src/features/timeline/Timeline.tsx`,
`apps/surf-chat-web/src/app/ChatShell.tsx`.

## 2026-06-15 — Дёрганье таймлайна при скролле → убрали виртуализацию

**Проблема.** Лента сообщений дёргалась при любом скролле.

**Причина.** `@tanstack/react-virtual` с оценкой высоты строки 78px при сообщениях
переменной (и большой) высоты: при скролле виртуализатор домерял реальные размеры
и пересчитывал позиции, контент прыгал. Чем больше сообщений в DOM — тем хуже.

**Решение.** Убрали виртуализацию — рендерим загруженные сообщения напрямую в DOM
(`.timeline__item` → `position: relative`). История грузится лениво, поэтому узлов
в DOM немного. Также добавили `overflow-anchor: none` на ленту, чтобы нативный
скролл-анкеринг браузера не конфликтовал с ручным позиционированием при подгрузке
истории. Если в будущем упрёмся в перф на тысячах сообщений — возвращаться к
виртуализации стоит на `react-virtuoso` (он создан под чат-кейс), а не на tanstack.

## 2026-06-15 — Telegram-style вход: заполнять экран историей

**Проблема.** При первом входе в чат было видно только ~10 последних сообщений
(окно начальной синхронизации Matrix), сверху пустота; история подгружалась
только при ручном скролле вверх.

**Решение.** Пока вид прижат к низу и экран не заполнен, лента жадно догружает
историю (`runLoad({ keepBottom: true })`) — чат открывается уже заполненным, как в
Телеграме. Файл: `apps/surf-chat-web/src/features/timeline/Timeline.tsx`.
